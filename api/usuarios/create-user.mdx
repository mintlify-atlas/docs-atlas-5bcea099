title: "POST / — Create a New User in msvc-usuarios"
sidebarTitle: "Create User"
description: "Creates a new Usuario record. The password is BCrypt-encoded before storage and is never returned. Requires a Bearer token with the OAuth2 write scope."
---

Persists a new `Usuario` to the MySQL database. Before saving, the plaintext `password` is encoded using BCrypt — the raw value is never stored or returned. The `email` field must be unique across all users; submitting a duplicate email returns a `400 Bad Request` with a descriptive message. Bean Validation is also enforced on the request body, and any constraint violations are returned as a field-keyed error map. A valid OAuth2 Bearer token with the **`write`** scope is required.

**Base URL:** `http://localhost:8001`  
**Gateway URL:** `http://localhost:8090/api/usuarios`

---

## Endpoint

```
POST http://localhost:8001/
```

---

## Request

### Headers

| Header | Value |
|---|---|
| `Authorization` | `Bearer <access_token>` |
| `Content-Type` | `application/json` |

### Body Parameters

<ParamField body="nombre" type="string" required>
  The user's display name. Must not be blank.
</ParamField>

<ParamField body="email" type="string" required>
  A valid, unique email address. If a user with this email already exists the request is rejected with `400`.
</ParamField>

<ParamField body="password" type="string" required>
  The user's plaintext password. It will be BCrypt-encoded before being written to the database and is **never** returned in any response.
</ParamField>

---

## Response

### 201 Created

Returns the newly created `Usuario` object. The `password` field is **not** included in the response.

<ResponseField name="id" type="number">
  The auto-generated primary key assigned to the new user.
</ResponseField>

<ResponseField name="nombre" type="string">
  The user's display name as provided.
</ResponseField>

<ResponseField name="email" type="string">
  The user's unique email address as provided.
</ResponseField>

### 400 Bad Request — Validation Errors

Returned when one or more Bean Validation constraints are violated. The body is an object whose keys are the failing field names and whose values describe the error.

```json
{
  "nombre": "El campo nombre must not be blank",
  "email": "El campo email must not be blank",
  "password": "El campo password must not be blank"
}
```

### 400 Bad Request — Duplicate Email

Returned when the submitted `email` is already registered to an existing user.

```json
{
  "mensaje": "ya existe un usuario con este correo!"
}
```

## Example

### Request

```bash
curl -X POST http://localhost:8001/ \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Ana García",
    "email": "ana.garcia@example.com",
    "password": "s3cr3tP@ss"
  }'
```

Via the API Gateway:

```bash
curl -X POST http://localhost:8090/api/usuarios/ \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Ana García",
    "email": "ana.garcia@example.com",
    "password": "s3cr3tP@ss"
  }'
```

### Request Body

```json
{
  "nombre": "Ana García",
  "email": "ana.garcia@example.com",
  "password": "s3cr3tP@ss"
}
```

### 201 Response

```json
{
  "id": 3,
  "nombre": "Ana García",
  "email": "ana.garcia@example.com"
}
```

### 400 Response — Duplicate Email

```json
{
  "mensaje": "ya existe un usuario con este correo!"
}
```

### 400 Response — Validation Errors

```json
{
  "password": "El campo password must not be blank",
  "email": "El campo email must not be blank"
}
```
