---
title: "POST / — Create a New User in msvc-usuarios"
sidebarTitle: "Create User"
description: "Creates a new Usuario record. Password is BCrypt-encoded before storage and returned as a hash in the 201 response. Requires the OAuth2 write scope."
---

Persists a new `Usuario` to the MySQL database. Before saving, the plaintext `password` is encoded using BCrypt — the raw value is never stored. The `email` field must be unique across all users; submitting a duplicate email returns a `400 Bad Request` with a descriptive message. A valid OAuth2 Bearer token with the **`write`** scope is required.

**Base URL:** `http://localhost:8001`  
**Gateway URL:** `http://localhost:8090/api/usuarios`

---

## Endpoint

```
POST http://localhost:8001/
```

---

## Request

### Headers

| Header | Value |
|---|---|
| `Authorization` | `Bearer <access_token>` |
| `Content-Type` | `application/json` |

### Body Parameters

<ParamField body="nombre" type="string" required>
  The user's display name.
</ParamField>

<ParamField body="email" type="string" required>
  A valid, unique email address. If a user with this email already exists the request is rejected with `400`.
</ParamField>

<ParamField body="password" type="string" required>
  The user's plaintext password. It will be BCrypt-encoded before being written to the database. The BCrypt hash is returned in the response body.
</ParamField>

---

## Response

### 201 Created

Returns the newly created `Usuario` object, including the BCrypt-encoded password hash.

<ResponseField name="id" type="number">
  The auto-generated primary key assigned to the new user.
</ResponseField>

<ResponseField name="nombre" type="string">
  The user's display name as provided.
</ResponseField>

<ResponseField name="email" type="string">
  The user's unique email address as provided.
</ResponseField>

<ResponseField name="password" type="string">
  The BCrypt-encoded password hash (e.g. `$2a$10$...`). The original plaintext password is never stored or echoed.
</ResponseField>

### 400 Bad Request — Duplicate Email

Returned when the submitted `email` is already registered to an existing user.

```json
{
  "mensaje": "ya existe un usuario con este correo!"
}
```

---

## Example

### Request

```bash
curl -X POST http://localhost:8001/ \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Ana García",
    "email": "ana.garcia@example.com",
    "password": "s3cr3tP@ss"
  }'
```

Via the API Gateway:

```bash
curl -X POST http://localhost:8090/api/usuarios/ \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Ana García",
    "email": "ana.garcia@example.com",
    "password": "s3cr3tP@ss"
  }'
```

### Request Body

```json
{
  "nombre": "Ana García",
  "email": "ana.garcia@example.com",
  "password": "s3cr3tP@ss"
}
```

### 201 Response

```json
{
  "id": 3,
  "nombre": "Ana García",
  "email": "ana.garcia@example.com",
  "password": "$2a$10$Xv3k8qW2mN5pL9oJ1rT6uO7yZ4hB0cD3eF5gH8iJ2kM4nP6qR8sU"
}
```

### 400 Response — Duplicate Email

```json
{
  "mensaje": "ya existe un usuario con este correo!"
}
```
